Banco de dados


O que é um SGBD? Tipos e Aplicações


Um SGBD (Sistema de Gerenciamento de Banco de Dados) é um software responsável por criar, administrar, organizar e controlar o acesso a bancos de dados. Ele permite armazenar informações de forma estruturada e segura, garantindo que usuários e sistemas possam inserir, consultar, atualizar e excluir dados de maneira eficiente.


Existem vários tipos de SGBDs, como os relacionais (que usam tabelas), os NoSQL (voltados para dados mais flexíveis), os hierárquicos, os orientados a objetos e os SGBDs em nuvem. Cada tipo atende necessidades diferentes, desde sistemas simples até aplicações de grande escala.


Quais os Principais Bancos de Dados?


Os bancos de dados mais utilizados no mercado são aqueles que combinam desempenho, segurança e facilidade de integração com sistemas modernos. Entre os relacionais, destacam-se MySQL, PostgreSQL, Oracle, SQL Server e MariaDB. Eles são amplamente usados por empresas, sites e aplicativos que necessitam de tabelas organizadas e consultas estruturadas.



Entre os bancos NoSQL, os mais populares são MongoDB, Firebase, Cassandra e Redis. Eles são ideais para aplicações que precisam lidar com grandes volumes de dados, informações não estruturadas ou atualizações rápidas, como redes sociais, jogos online e sistemas distribuídos.


O que é um Dicionário de Dados e Tipos de Dados


O dicionário de dados é um conjunto de informações que descreve todos os elementos presentes no banco de dados. Ele explica o significado dos campos, os tipos de dados utilizados, relacionamentos, restrições e regras de negócio. É essencial para documentação e desenvolvimento de sistemas bem estruturados.


Os tipos de dados mais comuns em um banco incluem números inteiros (INT), números decimais (FLOAT, DECIMAL), textos curtos (VARCHAR), textos longos (TEXT), datas e horários (DATE, TIME, DATETIME), valores booleanos (TRUE/FALSE) e tipos especiais como JSON, BLOB e ENUM.

Modelagem de Banco de Dados, DER e MER


A modelagem de banco de dados é o processo de planejar e organizar como as informações serão armazenadas. Através dela, criamos estruturas lógicas e coerentes que facilitam o entendimento e a manutenção do sistema. Uma boa modelagem reduz erros, evita redundâncias e melhora o desempenho.


O DER (Diagrama Entidade-Relacionamento) é uma representação visual das entidades do sistema e dos relacionamentos entre elas. Já o MER (Modelo Entidade-Relacionamento) descreve essas entidades de forma mais detalhada, definindo atributos, cardinalidade e regras de associação.


Ambos são fundamentais no desenvolvimento de sistemas, pois auxiliam na criação do banco de dados final usado pelo SGBD.

O proprietário possui um relacionamento 1 para n com o imóvel, indicando que um proprietário pode ter vários imóveis. O Corretor se relaciona tanto com proprietários quanto com inquilinos em relações n para n, já que pode atender vários clientes e ser contactado por muitos deles.


O Imóvel possui uma relação 1 para n com o inquilino no processo de aluguel: cada imóvel é alugado para um inquilino por vez, mas um inquilino pode alugar vários imóveis. Assim, o DER organiza como proprietários, corretores, imóveis e inquilinos interagem dentro do sistema de uma imobiliária.



Diagrma MER explicado



O diagrama apresenta três entidades principais: Médico, Paciente e o relacionamento Atendimento. A entidade Médico possui dois atributos: nome e código_CRM, que identificam e caracterizam o profissional. A entidade Paciente também possui atributos próprios: nome, CPF e Telefone (que está representado como atributo multivalorado, pois o paciente pode ter mais de um número). Além disso, o paciente possui um atributo composto chamado Endereço, dividido em Rua, Número e Bairro.


O relacionamento Atendimento liga o médico ao paciente e possui um atributo próprio: data, indicando quando o atendimento ocorreu. Assim, cada atendimento é um evento que envolve um médico que atende um paciente em uma determinada data.
